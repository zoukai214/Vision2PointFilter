# 点云语义标签回写设计

## 背景

当前项目已经具备以下能力：

- 对原始 LiDAR 点云执行 deskew，并导出新的 PCD。
- 按时间戳匹配前视广角图像。
- 将点云投影到图像上生成可视化结果。

本次需要新增的能力是：基于已经完成语义分割的单通道投影图，为每个导出点补充语义类别字段。图像中每个像素的灰度值表示一个语义类别，灰度值与类别编号的对应关系来自仓库根目录的 `class_to_grayscale_mapping_panoptic.json`。最终导出的 PCD 需要新增字段 `semantic_label`，其值为映射文件中的 `contiguous_id`。

## 目标

- 在导出的 PCD 中新增字段 `semantic_label`。
- `semantic_label` 的值使用 `contiguous_id`。
- 输入点类型保持不变，避免影响原始 PCD 加载路径。
- 语义缺失时不删点，使用约定值 `-1` 标记。

## 非目标

- 不修改原始输入 PCD 的字段定义。
- 不新增图像目录或映射文件的配置项。
- 不改变当前投影可视化 PNG 的行为。
- 不在本次设计中引入类别过滤、点云裁剪或统计分析功能。

## 总体方案

采用“新增导出点类型”的方案：

- 保留现有 `GacPcdPoint` 作为输入和 deskew 过程中的内部点类型。
- 新增一个导出专用点类型，例如 `SemanticLabeledPoint`，字段包含当前导出所需字段以及新增的 `int semantic_label`。
- 新增语义标注模块，负责解析 `class_to_grayscale_mapping_panoptic.json`，并提供 `gray_value -> contiguous_id` 查表能力。
- 在 `deskew_clip_export` 中对 deskew 后的点重新投影到匹配到的分割图像，从像素灰度值得到语义类别，并写入导出点类型后保存为 PCD。

这样可以把“输入数据结构”和“输出增强格式”隔离，降低对现有加载、投影和测试代码的影响。

## 模块设计

### 1. 导出点类型

新增导出专用点类型，建议放在 `src/data_loader/` 或与导出流程紧邻的位置，字段包括：

- `x`
- `y`
- `z`
- `intensity`
- `ring`
- `point_time_offset`
- `semantic_label`

其中 `semantic_label` 类型为 `int`，默认值为 `-1`。

设计原则：

- 字段布局尽量与现有 `GacPcdPoint` 保持一致，只在末尾增加 `semantic_label`。
- 仅在导出阶段使用该类型，避免影响原始点云读取。

### 2. 语义映射加载模块

新增独立模块，职责仅包括：

- 读取 `class_to_grayscale_mapping_panoptic.json`
- 校验 `mapping` 数组中的 `gray_value` 和 `contiguous_id`
- 构建 `gray_value -> contiguous_id` 的查表结构
- 提供按灰度值查询语义标签的接口

建议接口特征：

- 加载失败时返回明确错误
- 查询未命中时返回失败，由调用方决定使用 `-1`

实现要求：

- 只支持当前 JSON 结构，不做泛化配置扩展
- 如果 JSON 中出现重复的 `gray_value`，应视为配置错误并终止启动

### 3. 语义赋值流程

在 `deskew_clip_export` 中引入语义赋值步骤，处理顺序如下：

1. 读取并解析配置
2. 构建图像索引
3. 加载灰度映射 JSON
4. 读取原始点云并完成 deskew
5. 为每个 deskew 后的点执行投影
6. 如果投影像素有效，则读取单通道图像该像素的灰度值
7. 使用灰度映射查得 `contiguous_id`
8. 将结果写入导出点的 `semantic_label`
9. 保存带 `semantic_label` 的 PCD

其中第 5 步到第 8 步按点执行，不改变当前点的保留策略。

## 投影与语义规则

### 图像要求

- 使用当前 `projection.image_subdir` 对应目录中的 PNG 图像。
- 图像应为单通道灰度图。

如果图像不是单通道灰度图，本次实现应将其视为错误并终止当前运行。原因是本功能的输入语义已明确依赖“灰度值表示类别”，静默兼容彩色图会掩盖数据问题。

### 语义标签规则

- 命中有效像素且灰度值能在映射表中查到时，写入对应 `contiguous_id`
- 点投影失败时，写入 `-1`
- 投影落在图像外时，写入 `-1`
- 像素灰度值未在映射表中定义时，写入 `-1`

### 点数保持规则

语义赋值不会改变 deskew 后点云的点数。即使点没有有效语义，也仍然导出，只是 `semantic_label = -1`。

## 错误处理

以下场景应作为运行错误直接失败：

- `class_to_grayscale_mapping_panoptic.json` 不存在或无法解析
- JSON 缺少 `mapping`、`gray_value`、`contiguous_id` 等关键字段
- 映射表中存在重复 `gray_value`
- 匹配到的分割图像无法读取
- 匹配到的图像不是单通道灰度图

以下场景不应导致运行失败，只影响对应点的 `semantic_label`：

- 单个点无法投影
- 单个点投影后落在图像外
- 单个像素灰度值在映射表中未定义

## 代码落点

建议修改范围如下：

- `src/data_loader/`
  - 新增导出点类型定义头文件
- `src/projection/`
  - 新增语义灰度映射加载与查询模块
  - 如有必要，抽取通用“点到像素”计算接口，供渲染和语义赋值共用
- `src/application/deskew_clip_export.cc`
  - 初始化语义映射
  - 在导出前执行按点语义赋值
  - 输出带 `semantic_label` 的 PCD

其中投影几何逻辑应尽量复用现有实现，避免在渲染和语义赋值中重复维护两套像素计算代码。

## 测试设计

本次新增测试分为两类。

### 1. 映射加载测试

验证内容包括：

- 能正确解析最小合法 JSON
- 已知 `gray_value` 能映射到正确 `contiguous_id`
- 未知 `gray_value` 查询失败
- 重复 `gray_value` 会被拒绝

### 2. 语义投影测试

构造小尺寸相机模型、少量测试点和单通道测试图像，验证：

- 可投影点能得到正确 `semantic_label`
- 投影失败的点写入 `-1`
- 图像内但灰度值未知的点写入 `-1`

测试目标是把“映射解析”和“像素语义赋值”与现有投影渲染测试解耦，避免一个测试同时承担可视化和语义逻辑验证。

## 风险与控制

### 风险 1：投影逻辑重复

如果语义赋值直接在 `deskew_clip_export` 内复制当前投影数学逻辑，后续一旦相机投影模型调整，容易出现渲染结果和语义结果不一致。

控制策略：

- 抽取共享的点到像素计算逻辑，渲染和语义赋值共用。

### 风险 2：图像格式不一致

如果分割图被错误生成为三通道或调色板图，直接读取像素值可能得到错误语义。

控制策略：

- 明确要求输入图像为单通道灰度图，不满足时启动失败。

### 风险 3：未知灰度值污染结果

如果分割图中出现映射表外的灰度值，语义结果可能不完整。

控制策略：

- 单点级别使用 `-1` 兜底
- 运行时可记录未命中灰度值，便于后续排查数据问题

## 验收标准

- 导出的 PCD 中包含字段 `semantic_label`
- 对命中分割图有效像素且灰度值合法的点，`semantic_label` 为正确的 `contiguous_id`
- 对无有效语义的点，`semantic_label = -1`
- 不影响现有 deskew 主流程和投影 PNG 输出
- 新增对应单元测试并通过
