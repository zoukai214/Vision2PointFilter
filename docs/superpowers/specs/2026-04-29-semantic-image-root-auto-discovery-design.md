# 语义图根目录自动发现设计

## 1. 目标

当前配置 `projection.image_subdir` 使用值 `images_seg_mask2former/front_wide`，语义上表示单路相机目录。

本次改造目标是将该配置项的语义调整为语义图根目录，配置值改为 `images_seg_mask2former`。程序启动后按固定相机名列表自动发现其下子目录，并将这些子目录作为多路语义图输入来源。

本次设计不兼容旧语义，不保留 `images_seg_mask2former/front_wide` 作为正式输入方式。

## 2. 范围与非目标

### 2.1 范围

- 调整 `projection.image_subdir` 的配置语义
- 程序按固定 7 路相机名自动拼接并发现语义图子目录
- 将当前单路 `front_wide` 图像索引和语义查询流程提升为多路相机流程
- 保持现有投影公式、时间匹配规则、灰度映射规则不变
- 补充对应单元测试

### 2.2 非目标

- 不兼容旧的单路目录配置语义
- 不修改运动补偿逻辑
- 不引入多路投票或置信度融合
- 不修改 CMake 编译选项

## 3. 固定业务规则

### 3.1 固定相机名列表

程序内置固定相机顺序：

1. `front_wide`
2. `back`
3. `front_narrow`
4. `left_back`
5. `left_front`
6. `right_back`
7. `right_front`

该顺序同时用于：

- 语义图子目录发现
- 标定加载
- 单点语义查询优先级

### 3.2 目录发现规则

当配置为：

```yaml
projection:
  image_subdir: images_seg_mask2former
```

程序按以下规则为每一路相机拼接语义图目录：

`<clip_root>/<image_subdir>/<camera_name>`

例如：

- `.../images_seg_mask2former/front_wide`
- `.../images_seg_mask2former/back`
- `.../images_seg_mask2former/front_narrow`

若 `image_subdir` 为绝对路径，则按 `<image_subdir>/<camera_name>` 拼接。

### 3.3 失败规则

以下任一情况发生时，程序直接失败：

- 某一路相机目录不存在
- 某一路相机目录无法建立图像时间索引
- 某一路相机标定加载失败
- 某一路匹配到的语义图无法读取或图像类型不合法

错误日志必须包含相机名和具体目录或文件路径，便于排查数据问题。

## 4. 运行时流程

### 4.1 启动阶段

启动时执行以下步骤：

1. 读取 `projection.image_subdir`
2. 根据固定 7 路相机名生成 7 个语义图目录
3. 为每一路相机加载标定
4. 为每一路相机建立图像时间索引
5. 构建多相机投影与语义查询上下文

启动阶段不再只构建 `front_wide` 单路上下文。

### 4.2 每帧处理阶段

对每一帧点云：

1. 为每一路相机查找与当前点云帧时间戳最近的语义图
2. 校验时间差是否在 `projection.max_time_diff_ms` 允许范围内
3. 读取对应语义图
4. 对每个点按固定相机优先级依次执行查询

### 4.3 单点语义查询规则

对单个点按相机优先级执行：

1. 投影到当前相机
2. 若投影失败，继续下一路
3. 若像素越界，继续下一路
4. 若灰度值无法映射为语义标签，继续下一路
5. 若命中有效语义标签，立即返回该标签
6. 若所有相机均未命中，保持未标注值 `-1`

该流程中：

- “未命中标签”属于正常业务结果，不是函数错误
- “输入缺失、图像类型错误、必要对象为空”属于执行错误

## 5. 接口与结构调整

### 5.1 路径接口

当前 `GacClipRootLoader::FrontWideImageDir(...)` 的接口名与语义绑定了单路相机。

本次应将其泛化为：

- 接收语义图根目录
- 接收相机名
- 返回目标相机的语义图目录

接口命名不应再包含 `front_wide`，避免继续固化单路语义。

### 5.2 配置语义

当前默认配置：

```yaml
projection:
  image_subdir: images_seg_mask2former/front_wide
```

调整为：

```yaml
projection:
  image_subdir: images_seg_mask2former
```

字段名暂不修改，避免不必要的 YAML 结构变更；但字段语义正式更新为“语义图根目录”。

### 5.3 多相机上下文

当前运行时上下文只包含：

- 单个相机模型
- 单个图像索引
- 单个输出目录配置

本次提升为按相机名组织的集合结构，用于持有：

- 相机模型
- 图像索引
- 当前帧匹配语义图

几何投影公式本身保持不变，只提升输入组织方式。

## 6. 测试设计

必须补充以下测试：

### 6.1 配置与路径测试

- 当 `image_subdir` 配置为 `images_seg_mask2former` 时，能够为 7 路相机生成正确目录
- 相对路径输入时，目录基于 `clip_root` 解析
- 绝对路径输入时，目录不再拼接 `clip_root`

### 6.2 语义查询优先级测试

- 当前高优先级相机未命中、低优先级相机命中时，返回低优先级命中的标签
- 高优先级相机命中时，不再继续查询后续相机

### 6.3 回归测试

- 当仅 `front_wide` 命中时，结果与当前单路逻辑保持一致

### 6.4 失败测试

- 缺少任一路相机目录时，初始化失败
- 任一路相机图像索引构建失败时，初始化失败
- 语义图不是单通道灰度图时，处理失败

## 7. 风险与控制

主要风险：

- 旧配置文件继续使用 `images_seg_mask2former/front_wide` 会被解释为根目录，从而导致子目录拼接错误
- 单相机接口泛化为多相机集合后，容易遗漏某些日志和错误分支中的相机名

控制方式：

- 同步更新默认配置文件和相关测试输入
- 所有初始化失败日志带相机名、目录路径、文件路径
- 保留 `front_wide` 单路命中回归测试，防止行为退化

## 8. 结论

本次改造将 `projection.image_subdir` 正式定义为语义图根目录，并按固定 7 路相机名自动发现子目录，不兼容旧的单路目录语义。实现上保持现有投影和语义映射算法不变，只将输入组织方式从单相机提升为多相机集合，并通过单元测试覆盖目录解析、优先级查询、回归行为和失败路径。
